# Установка через мастер (Unity Hub Pro)

Данная инструкция описывает установку Unity Open MCP с помощью Unity Hub Pro.

## Для кого это

Это способ установки — без терминала, без ручного редактирования JSON, без
Git-URL. То есть, если не хотите открывать командную строку, то эта инструкция для вас.

Для пути «сделай сам» (без приложения Hub) см. [manual-setup.md](manual-setup.md).
Хотите, чтобы установку выполнил ИИ-агент? См. [agent-setup.md](agent-setup.md).

## Требования

- **Unity 2022.3 LTS или новее** (рекомендуется Unity 6).
- **Node.js 18 или новее** — нужен только потому, что MCP-сервер — небольшая
  Node-программа. Установите с <https://nodejs.org/> (кнопка **LTS**), если у вас
  её нет. Мастер это проверит и сообщит, если отсутствует или устарела.
- **Unity Hub Pro** — сначала установите. См. [unity-hub-pro.md](../../unity-hub-pro.md)
  (скачайте установщик для вашей ОС со страницы GitHub Releases).
- **MCP-клиент** (Cursor, Claude Desktop, Claude Code, OpenCode, ZCode, Cline,
  Codex, VS Code Copilot, Gemini CLI или другой поддерживаемый агент) — ИИ-инструмент,
  из которого вы будете управлять Unity.

## Общий поток

1. **Установите Unity Hub Pro**, если ещё не установили (см. [unity-hub-pro.md](../../unity-hub-pro.md)).
2. Откройте Unity Hub Pro и добавьте ваш проект Unity.
3. Нажмите действие **AI** для этого проекта. Кнопка становится **зелёной**, когда
   агент уже установлен и настроен для этого проекта; в противном случае она
   янтарная/синяя и открывает мастер.
4. Выберите пресет настройки (или **Custom / skip**) на шаге пресетов.
5. Пройдите мастер — пресет Recommended может сделать всё одним кликом через
   **Express setup**.
6. Перезапустите MCP-клиент.
7. Выполните вызов Unity MCP для проверки соединения.

Выборы в мастере (пресет, MCP-клиент, переключатели пакетов, порт моста и другие
поля формы) запоминаются **для каждого проекта** при повторном открытии мастера;
вы всегда начинаете снова с выбора пресета.

## Шаги мастера

Мастер показывает вверху кликабельную полосу прогресса. Сегменты становятся
**зелёными**, когда их проверки уже проходят, поэтому с одного взгляда видно, какие
шаги ещё требуют внимания. Вы можете кликнуть любой сегмент, чтобы перейти к этому
шагу; Back/Next по-прежнему двигаются последовательно. Рекомендуемый путь не
требует раскрытия **Advanced (optional)** ни на одном шаге.

### Preset — пресет настройки (опционально)

Выберите пресет, чтобы предзаполнить остальную часть мастера, или **Custom / skip**,
чтобы настраивать каждый шаг вручную. Пресеты — отправные точки, а не ограничения:
любое поле можно изменить на последующих шагах. Первый экран показывает три частых
варианта; нишевые пресеты — за **More presets**.

| Пресет | Лучше всего для | Предзаполняет |
|---|---|---|
| **Regular user (npm)** *(рекомендуется)* | Разработчики, желающие опубликованный npm-пакет без чекаута монорепозитория | `npx -y unity-open-mcp@0.7.0`; bridge + verify из опубликованных источников; доменные зависимости выключены; скил включён |
| **Contributor (local checkout)** | Контрибьюторы монорепозитория, правящие bridge / verify / MCP-сервер | Локальный чекаут + пакеты `file:` из клона; доменные зависимости выключены; скил включён. Сначала соберите `mcp-server/` (см. [Установку для разработки](development-setup.md)) |
| **Custom / skip** | Все, кто хочет встроенные умолчания мастера | Без предзаполнения — идентично ручному потоку |

**More presets** (за раскрытием):

| Пресет | Лучше всего для | Предзаполняет |
|---|---|---|
| **Team CI** | Автоматизация на headless CI | Глобальная установка npm; клиент **Manual / CLI snippet**; скил пропущен; настройте токен-аутентификацию на мосту для CI |
| **Secure / remote** | Не-localhost доступ к мосту с ограниченными изменениями | Опубликованные источники; скил включён. Токен-аутентификация, удалённый bind и ограниченные группы инструментов — это контроли на стороне моста; настраивайте их из окна моста после подключения |

### Preflight — ворота окружения

Preflight проверяет ваше окружение и является воротами для всего остального. Проверки
делятся на две группы:

- **Blocking — должны пройти для продолжения:** корректная раскладка проекта Unity,
  версия Unity (минимум 2022.3 LTS; рекомендуется Unity 6+), Node.js 18+ и
  перезаписываемый `Packages/manifest.json`. Все они должны пройти, прежде чем Next
  станет активен.
- **Setup status — обрабатываются на последующих шагах:** установлены ли пакеты
  bridge / verify, конфигурация MCP-клиента и агентский скил. Они читаются как
  «Not yet» (не как ошибки) и зеленеют по мере прохождения последующих шагов.

Одна кнопка **Re-check** повторно запускает обнаружение проекта и пробу Node.
Обнаружение и проба Node выполняются вне UI-потока и ограничены таймаутами, поэтому
медленный диск или зависший `node --version` выдаёт реальную ошибку вместо
зависания мастера. Вы всегда можете закрыть мастер через **Cancel**, **Escape** или
кнопку **×** — обнаружение остановится в фоне.

**Уже настроено?** Если bridge, verify и MCP-клиент уже настроены для проекта,
Preflight предлагает баннер **You're ready** с ярлыком **Go to Verify**, который
пропускает шаги применения.

**Express setup:** когда проверки окружения проходят, Preflight предлагает
**Express setup** — один клик **Set up** последовательно выполняет установку пакетов
→ запись конфигурации MCP-клиента → запуск/проверку с живым списком прогресса.
Полный пошаговый путь остаётся доступным через полосу прогресса.

### MCP server source (опционально / дополнительно)

Выберите, как запускается сервер `unity-open-mcp`. По умолчанию (`npx`) здесь
настройка не требуется, и мастер автоматически пропускает этот шаг на пути
Recommended — вы попадаете сюда только через полосу прогресса или пресет
**Custom / skip** / **Contributor**.

- **npx (published npm)** *(по умолчанию)* — извлекает последний `unity-open-mcp`
  из npm при первом запуске. Клон репозитория не нужен.
- **Global install** — запускает бинарник `unity-open-mcp` (предполагается
  `npm i -g unity-open-mcp`). Стабильный путь для CI-образов.
- **Local checkout** — указывает на клонированный монорепозиторий
  `unity-open-mcp`. Пакеты и копирование скила используют корень инструментария.
- **Custom entrypoint (advanced)** — конкретный путь `mcp-server/dist/index.js`
  для сборок вне корня инструментария.

Выберите один источник; показываются только поля для вашего выбора. Local checkout
и custom entrypoint оба требуют проверенного корня инструментария. Если используете
local checkout, сначала соберите:

```bash
cd mcp-server
npm install
npm run build
```

### Unity packages

Установите или обновите пакеты bridge и verify в `Packages/manifest.json`.
Обязательные переключатели — в секции по умолчанию; предпросмотр diff в реальном
времени пересчитывается при изменениях.

**Advanced (optional)** (свёрнуто по умолчанию — рекомендуемый путь никогда не
нуждается в нём):

- **Use local packages** — установка через пути `file:` относительно корня
  инструментария (типично для проектов внутри монорепозитория).
- **Package version pin** — переопределяет тег, к которому привязаны оба пакета.
  Оставьте пустым, чтобы установить версию, совпадающую со сборкой Hub.
- **Custom git URL** — заменяет git-remote корня инструментария (для тестирования
  против fork-а).
- **Unity domain dependencies (optional)** — NavMesh, Input System, ProBuilder.
  Доменные инструменты **поставляются с мостом**; они компилируются автоматически,
  как только появляется соответствующий пакет Unity. Отметьте те, которые мастер
  должен добавить. Встроенные модули (Particle System, Animation) идут в редакторе
  и не требуют записи в манифесте.

После подключения вы можете добавлять и удалять доменные зависимости Unity в любое
время из окна моста (**Tools → Unity Open MCP Bridge → Extensions → Optional Unity
dependencies**) — по одному клику на домен, без редактирования манифеста. Модальное
окно настроек проекта Hub также показывает статус «установлено/отсутствует» по
каждому домену только для чтения.

По рабочему процессу `file:` для контрибьютора / community-пака см.
[Установку для разработки](development-setup.md).

### Configure AI client

Выберите ИИ-клиент для подключения. Первый экран показывает короткий список
**Popular** (Cursor, Claude Desktop, VS Code Copilot, Claude Code, Manual); полный
каталог — за **Show all clients** с окном поиска.

Каждый вариант показывает, записывает ли он файл конфигурации, является ли
CLI-only или копирует сниппет, с целевым путём и форматом в его тултипе. Полный
каталог путей и оболочек — в
[Конфигурации MCP-клиента](client-configuration.md).

Просмотрите предпросмотр сгенерированной конфигурации (JSON или TOML, или команда
CLI для Claude Code), затем запишите. Запись безопасна для слияния: посторонние ключи
и соседние MCP-серверы сохраняются, а рядом с оригинальным файлом остаётся резервная
копия `.bak`.

**Advanced (optional):** переопределение **Bridge HTTP port** находится здесь
(свёрнуто по умолчанию; порт автоматически выводится из пути проекта).

### Agent skill (опционально)

Агентский скил даёт вашему ИИ-клиенту руководство по рабочим процессам для
инструментов Unity MCP — цикл mutate→gate→fix, приоритетное обнаружение через
capabilities и агентские «чувства» (тесты, profiler, скриншоты). Две опции пишут в
одну и ту же относительную для проекта папку(и) скила для выбранного клиента
(ZCode → `.agents/skills/`, Cursor → `.cursor/skills/` и т. д.):

- **Copy skill** — устанавливает шаблонный плейбук (`skills/unity-open-mcp/SKILL.md`).
  Одно и то же руководство по рабочему процессу для каждого проекта; сборка не требуется.
- **Generate project skill** — создаёт специфичный для проекта `SKILL.md`,
  объединяющий шаблонный плейбук с инвентарём этого проекта (версия Unity,
  установленные пакеты, ключевые типы MonoBehaviour / ScriptableObject). Требует
  собранного MCP-сервера (`mcp-server/dist/index.js`).

Оба подчиняются явному чекбоксу перезаписи; существующие файлы резервируются в
`*.bak` перед заменой. Можно только копировать, только генерировать или оба
действия (генерация перезаписывает тот же путь, что и копирование, поэтому
подтвердите перезапись).

Пресет **Team CI** автоматически пропускает этот шаг — CI-агентам обычно не нужен
файл десктопного скила.

### Launch and verify

- Запустите Unity
- Дождитесь компиляции и готовности моста
- Завершите, когда проверки работоспособности пройдут

Во время ожидания MCP-сервер автоматически закрывает распространённые стартовые
модальные окна Unity (Safe Mode, несовпадение версий и подобные) согласно
`UNITY_OPEN_MCP_DIALOG_POLICY`. Если этот шаг зависает на модальном окне, см.
[Политику диалогов](../../dialog-policy.md).

## Clear AI Setup

В подвале мастера есть жёлтая кнопка **Clear AI Setup** (справа внизу). Она
удаляет все артефакты, записанные мастером для текущего проекта, после запроса
подтверждения:

- записи bridge + verify из `Packages/manifest.json`
- запись `unity-open-mcp` из каждой известной конфигурации MCP-клиента
  (конфигурации уровня проекта безусловно; глобальные — только ту запись, чей путь
  проекта совпадает с этим)
- скопированные файлы агентского скила `SKILL.md`

Рядом с каждым изменённым файлом создаётся резервная копия `.bak`. Ошибки по
отдельным целям сообщаются инлайн, а не прерывают весь проход. Это действие нельзя отменить.

## Устранение неполадок

Перед продолжением устраните все блокирующие проверки Preflight. После установки
перезапустите клиент, убедитесь, что Unity открыл выбранный проект, и дождитесь
компиляции. **Re-check** обновляет состояние установки на диске; достижимость
моста проверяется только во время Launch and verify. Для процедур подключения и
восстановления используйте [Устранение неполадок](../../troubleshooting.md); для
стартовых модальных окон — [Политику диалогов](../../dialog-policy.md).
